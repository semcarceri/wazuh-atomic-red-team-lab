# Wazuh Endpoint Detection Lab Setup

This document outlines the setup of a Windows endpoint integrated with **Wazuh**, **Sysmon**, and **Atomic Red Team** for security monitoring, detection engineering, and MITRE ATT&CK technique validation.

---

# 1. Wazuh Agent Installation

The Wazuh agent was installed locally on the Windows endpoint using the enrollment command generated by the Wazuh Manager.

## Installation Workflow

1. Generated the Windows agent enrollment command from the **Wazuh Manager**.
2. Copied the generated command into PowerShell on the endpoint.
3. The agent:
   - Downloaded automatically
   - Installed successfully
   - Registered with the manager

## Verify Installation

```powershell
Get-Service WazuhSvc
```

Successful output confirms that the endpoint is enrolled and capable of forwarding Windows telemetry to the Wazuh Manager.

---

# 2. Sysmon Installation

Sysmon was installed to provide high-fidelity endpoint telemetry required for detection engineering.

## Purpose

Sysmon improves visibility into:

- Process creation
- Command-line arguments
- Parent/child process relationships
- Network connections
- Registry modifications
- File creation
- Process access and injection

---

## Installation Steps

Install Sysmon using the Sysinternals binary:

```powershell
.\Sysmon64.exe -accepteula -i
```

## Verify Sysmon Service

```powershell
Get-Service Sysmon64
```

---

## Validate Sysmon Telemetry

Check generated Sysmon events:

```powershell
Get-WinEvent -LogName "Microsoft-Windows-Sysmon/Operational" -MaxEvents 10
```

# 3. Atomic Red Team Installation

Atomic Red Team was used to simulate adversary behavior mapped to MITRE ATT&CK techniques.

---

## Installing Invoke-AtomicRedTeam

Install required PowerShell modules:

```powershell
Install-Module Invoke-AtomicRedTeam,powershell-yaml -Scope CurrentUser
```

Install Atomic Red Team:

```powershell
IEX (IWR 'https://raw.githubusercontent.com/redcanaryco/invoke-atomicredteam/master/install-atomicredteam.ps1' -UseBasicParsing)

Install-AtomicRedTeam -GetAtomics
```

---

## Atomic Red Team Directory Structure

Expected installation layout:

```
C:\AtomicRedTeam\
│
├── atomics
│
├── ExternalPayloads
│
└── Invoke-AtomicRedTeam
```

# 4. Troubleshooting

## PowerShell Execution Policy Blocked

### Problem

Error:

```
powershell-yaml.psm1 cannot be loaded because running scripts is disabled
```

### Fix

Allow script execution:

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned

Set-ExecutionPolicy -Scope Process Bypass

Import-Module Invoke-AtomicRedTeam
```

---

# Missing Atomic Test Definitions

## Problem

Running:

```powershell
Invoke-AtomicTest T1059.001
```

Returned:

```
Cannot find path C:\AtomicRedTeam\atomics
```

---

## Fix

1. Download Atomic Red Team repository manually.
2. Extract:

```
C:\AtomicRedTeam\atomics\atomic-red-team-master\atomics
```

3. Specify the correct atomics path:

```powershell
Invoke-AtomicTest T1059.001 `
-PathToAtomicsFolder "C:\AtomicRedTeam\atomics\atomic-red-team-master\atomics"
```

---

# Missing Payload Scripts

## Problem

Error:

```
GetToken.ps1 is not recognized
```

while running:

```
T1134.002 - Create Process with Token
```

---

## Fix

Locate the missing script:

```powershell
Get-ChildItem C:\AtomicRedTeam -Recurse -Filter GetToken.ps1
```

Correct location:

```
C:\AtomicRedTeam\atomics\T1134.002\src\GetToken.ps1
```

Set the correct Atomic path:

```powershell
$PathToAtomicsFolder="C:\AtomicRedTeam\atomics"
```

# Sysmon Installation Misassumption

## Problem

Attempted:

```powershell
Enable-WindowsOptionalFeature -Online -FeatureName Sysmon
```

---

## Cause

Sysmon is **not** a Windows Optional Feature.

---

## Fix

Install Sysmon manually using Microsoft Sysinternals:

```powershell
.\Sysmon64.exe -accepteula -i
```

---

# Sysmon Logs Not Appearing in Wazuh

## Problem

Sysmon events were generated locally but not visible in Wazuh.

---

## Root Cause

The Wazuh agent was not configured to ingest the Sysmon Operational event channel.

---

## Fix

Edit:

```
C:\Program Files (x86)\ossec-agent\ossec.conf
```

Add:

```xml
<localfile>
  <location>Microsoft-Windows-Sysmon/Operational</location>
  <log_format>eventchannel</log_format>
</localfile>
```

Restart the Wazuh agent:

```powershell
Restart-Service WazuhSvc
```
